Now let’s go over React Lifecycle methods so we can determine if we should refactor the code above or not:

React Lifecycle Methods

componentWillMount()

componentWillMount is called before the component is rendered for the first time on the page.  Setting state in component will mount will not retrigger render so this is a good place to fetch data from your database.

componentDidMount()

componentDidMount is called one time after the component is initally loaded onto the page.  Setting state in componentDidMount is not reccomended as will will trigger an additional render that is not needed.  componentDidMount is used for interacting with the DOM after the component is on the page.  A good example is initializing some materialize JavaScript like:
componentDidMount() {
$(".button-collapse").sideNav();
}

componentWillRecieveProps()

Invoked when a component is recieving new props.  This is not called on initial render.  This is an opportunity to react when props have changed:
componentWillRecieveProps(nextProps) {
this.setState({ likesIncreasing: nextProps.likeCount > this.props.likeCount })
}

shouldComponentUpdate()


This must return a boolean.  By default it is always called and always returns true.  I don’t advise overwriting this method until you are a  lot more comfortable with react and state.

componentWillUpdate()

This is invoked immedately before rendering when shouldComponentUpdate returns a true.  NOTE:  You can’t set state in this function.

componentDidUpdate()

Much like componentDidMount except it is not called on initial render and is called on every update.

componentWillUnmount()

Invoked immediately before a component is unmounted.  This is used for any clean up task that need to happen.

These along with render are methods that come with React Components to control the lifecycle.

Lifecycle methods really only need to exist in a component managing state.  In React we usually deal with container components (Those containing state) and presentational components (Stateless “DUMB” components).  If we are not using state and we are not using lifecycle components then we really have a presentational component.  Looking at our App component above notice we are not using state and we are not using lifecycle methods.  So we can refactor this into a presentational component.  We actually prefer presentational components in react so it’s best to use them whenever you have the chance.

deleteProduct(id) {
event.preventDefault()
$.ajax({
url: `${BASE_URL}/products/${id}`,
type: 'DELETE'
}).done( () => {
let index = this.state.products.findIndex( product => product.id === id )
this.setState({
products: [
...this.state.products.slice(0, index),
...this.state.products.slice(index + 1, this.state.products.length)
]
})
})
}
render() {
return (

<Products products={this.state.products} deleteProduct={this.deleteProduct.bind(this)} />
<AddProduct />
</div>
)
}
Our delete product will take in an ID, make an api call to the server and when done it will setState to everything except for the item that we deleted.  Notice we are doing this in an immutable way.  Finally we pass deleteProduct as a prop down to Products.  We need to bind it because deleteProduct needs access to ‘this’.

import React from 'react'
import $ from 'jquery'
import { BASE_URL } from '../constants'
class AddProduct extends React.Component {
handleSubmit(e) {
e.preventDefault()
let name = this.refs.name.value
let description = this.refs.description.value
let base_price = this.refs.base_price.value
let quantity_on_hand = this.refs.quantity_on_hand.value
$.ajax({
url: `${BASE_URL}/products`,
type: 'POST',
data: { product: { name, description, base_price, quantity_on_hand } }
}).done( product => {
this.props.addProduct(product)
this.refs.addForm.reset()
})
}
render() {
return(
<div className="col m4">
<h4 className="center">Add Product</h4>
<form ref="addForm" onSubmit={this.handleSubmit.bind(this)}>

<input placeholder="description" ref="description" />
<input type="number" placeholder="base_price" ref="base_price" />
<input type="number" placeholder="quanitity" ref="quanity_on_hand" />
<button className="btn" type="submit">Add</button>
</form>
</div>
)
}
}
export default AddProduct
